02 / 日常操作

📱モニターの見方・使い方

ブックマークからモニターを開く
ページを開くと自動でデータ取得が始まる。15分ごとに自動更新。
ヘッダーの観測時刻を確認
「観測 HH:MM JST」が最新のFxEs観測時刻。古い場合はデータ未更新。
手動更新したい場合
右上の 更新 ボタンを押す。
アラートが出たら
画面上部に赤い点滅バナーが表示される。同時にLINEにもメッセージが届く。
マーカーの色の意味（FxEs）
灰色 = データなし　／　水色 = 通常　／　黄色 ≥ 7.0　／　ピンク ≥ 8.0　／　赤 ≥ 9.0 → アラート
03 / LINE トークン管理

🔑Channel Access Token とは

⚠ 最重要メンテナンス作業
LINE Messaging API の Channel Access Token には有効期限があります。
期限切れになると LINE通知が届かなくなります。定期的な更新が必要です。
トークンの種類
短期トークン
有効期限 30日
自動で失効する

長期トークン（ステートレス）
有効期限なし
ただし上限数あり（30個）
現在の設定
発行時に選んだ種類による。
LINE Developers コンソールで確認できる。

短期の場合は30日以内に更新が必要
確認方法
LINE Developers → チャンネル → Messaging API設定 → Channel access token
発行済みトークンの一覧と有効期限が表示される
04 / LINE トークン更新手順

🔄トークン更新の手順

LINE Developers を開く
https://developers.line.biz/console/ にアクセス
チャンネルを選択
「CB DX Alert」チャンネル → 「Messaging API設定」タブ
新しいトークンを発行
「Channel access token」の「Issue」または「再発行」ボタンを押す
新しいトークン文字列をコピーする
Cloudflare Workerに設定
https://dash.cloudflare.com/ → Workers & Pages → iono-line-alert
→ Settings → Variables and Secrets → LINE_TOKEN の編集ボタン → 新しい値を貼り付けて保存
動作確認
モニターページのDevTools Consoleでテスト送信（次のスライド参照）
05 / 動作テスト

🧪LINE通知テスト方法

モニターページを開く
ブックマークからCB DX Iono Monitorを開く
DevToolsを開く
PC: F12 または Ctrl+Shift+I（Mac: Cmd+Option+I）
「コンソール」タブを選択
「allow pasting」と入力してEnter
初回は貼り付け制限があるため、これを先に実行する
テストコードを貼り付けてEnter
fetch('https://iono-line-alert.yotsuzeki.workers.dev',{method:'POST',headers:{'Content-Type':'application/json'},body:JSON.stringify({message:'テスト通知'})}).then(r=>r.json()).then(console.log)
結果確認
コンソールに {ok: true, status: 200} と表示 → LINE受信を確認
06 / トラブル対応

🔧よくある問題と対処

症状 原因 対処
LINEが届かない トークン期限切れ LINE Developersで再発行 → Cloudflare LINE_TOKEN 更新
データが「--」のまま NICTまたはNOAAサーバー不通
またはCORSプロキシ障害		 しばらく待って「更新」ボタンを押す。数時間続く場合はNICTサイトを直接確認
観測時刻が古い NICTのデータ更新遅延 NICTの更新頻度によるため対処不要。数時間単位の遅延は正常
テストコードで ok: false Worker設定ミス Cloudflareで LINE_TOKEN と LINE_USER_ID を再確認。Workerを再デプロイ
アラートが出ない（9.0以上なのに） ブラウザのキャッシュ 「更新」ボタンを押す。または強制リロード（Ctrl+Shift+R）
